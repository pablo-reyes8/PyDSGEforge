# Security policy

## Supported versions

PyDSGEforge is currently pre-1.0. Security fixes are applied to the latest
revision on the default branch and included in the next release.

## Reporting a vulnerability

Please do not disclose suspected vulnerabilities in a public issue. Use
GitHub's **Security > Report a vulnerability** flow for this repository. If
private reporting is unavailable, contact the maintainer through the profile
linked from the repository and request a private channel without including
exploit details in the first message.

Include affected versions, environment, reproduction steps, potential impact,
and any suggested mitigation. You should receive acknowledgement within seven
days. Coordinated disclosure is preferred until a fix is available.

Numerical disagreements, malformed model specifications, and convergence
problems are normally correctness bugs rather than security issues; report
those through the public issue templates unless sensitive data are involved.
